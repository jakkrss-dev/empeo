======================================================
คู่มือระบบ Empeo Attendance Dashboard แบบ Cloud Automation 100%
======================================================

1. ภาพรวมของระบบ (System Overview)
-----------------------------------
ระบบนี้ถูกออกแบบมาเพื่อดึงข้อมูลเวลาเข้า-ออกงานจากระบบ Empeo โดยอัตโนมัติ 
และนำมาแสดงผลเป็นกราฟและสถิติผ่านหน้าเว็บ Dashboard โดยที่ไม่ต้องมีใครคอยเปิดคอมพิวเตอร์ทิ้งไว้
ทุกอย่างทำงานประสานกันผ่าน Cloud ทั้งหมด


2. หน้าที่ของแต่ละ Application (Components & Roles)
-----------------------------------
1) Next.js Dashboard (ฝั่งหน้าเว็บ / Frontend)
   - โฮสต์อยู่ที่: Vercel
   - หน้าที่: เป็นหน้าเว็บไซต์ที่แสดงสถิติและกราฟต่างๆ ให้ผู้ใช้เห็น
   - การทำงาน: เมื่อมีคนเปิดหน้าเว็บ มันจะไปดึงไฟล์ข้อมูลล่าสุดจาก GitHub Gist มาประมวลผลและแสดงผลทันที และมีปุ่มสำหรับส่งคำสั่ง (API) ไปปลุกให้ GitHub Actions ทำงาน

2) GitHub Actions (ฝั่งเซิร์ฟเวอร์บอท / Cloud Runner)
   - โฮสต์อยู่ที่: GitHub
   - หน้าที่: ทำหน้าที่เป็น "คอมพิวเตอร์จำลอง" บน Cloud ที่คอยรันบอทดึงข้อมูล 
   - การทำงาน: จะถูกตั้งเวลาให้ตื่นขึ้นมาทำงานทุกวันตอน 18:00 น. หรือตื่นขึ้นมาทำงานทันทีเมื่อได้รับคำสั่งจากปุ่มบนหน้าเว็บ Vercel 

3) Python Selenium Bot (ไฟล์ headless_empeo.py)
   - หน้าที่: เป็นสคริปต์ "บอท" ที่คอยจำลองการทำงานของคน
   - การทำงาน: 
      > เปิดเบราว์เซอร์ Chrome แบบซ่อนหน้าจอ (Headless)
      > ล็อกอินเข้าระบบ Empeo
      > เข้าไปที่หน้ารายงาน C009 
      > กดเลือก "สังกัด" เป็น "เลือกทั้งหมด"
      > กดปุ่ม "แสดง" และ "ดาวน์โหลด" ไฟล์ Excel
      > แปลงไฟล์ Excel เป็นข้อความรหัสลับ (Base64) แล้วส่งไปบันทึกทับใน GitHub Gist

4) GitHub Gist (ฝั่งฐานข้อมูล / Cloud Database)
   - โฮสต์อยู่ที่: GitHub Gist
   - หน้าที่: ทำหน้าที่เสมือน Database ขนาดย่อมๆ (ฟรี) เพื่อเป็นตัวกลางในการส่งผ่านข้อมูล
   - การทำงาน: รับไฟล์ Excel (ที่ถูกแปลงเป็น Base64) จาก Python Bot มาเก็บไว้ เพื่อรอให้ฝั่งหน้าเว็บ Next.js มาดึงข้อมูลเอาไปแสดงผล


3. ขั้นตอนการทำงานแบบเป็นลำดับ (Step-by-Step Workflow)
-----------------------------------
[กรณีที่ 1: การอัปเดตแบบอัตโนมัติตามเวลา (Cron Job)]
  1. เมื่อถึงเวลา 18:00 น. ของทุกวัน GitHub Actions จะเริ่มรันตัวเองอัตโนมัติ
  2. สคริปต์ headless_empeo.py จะเข้าไปโหลดไฟล์ Excel ล่าสุดจากเว็บ Empeo
  3. สคริปต์อัปโหลดข้อมูลไฟล์ทับลงใน GitHub Gist
  4. เช้าวันถัดไป เมื่อหัวหน้าเปิดหน้าเว็บ Vercel ข้อมูลก็จะเป็นของเมื่อวานเย็นโดยสมบูรณ์

[กรณีที่ 2: การอัปเดตแบบสั่งการด้วยตัวเอง (Manual Trigger)]
  1. ผู้ใช้กดปุ่ม "สั่งบอทอัปเดต (รอ 1 นาที)" บนเว็บ Vercel
  2. ระบบ Vercel ส่ง API ไปกระตุ้น (Repository Dispatch) ให้ GitHub Actions ตื่นขึ้นมาทำงาน
  3. GitHub Actions รันสคริปต์ headless_empeo.py ดึงข้อมูลจาก Empeo
  4. อัปโหลดข้อมูลใหม่ลง GitHub Gist (ใช้เวลาทำงานทั้งหมดประมาณ 1-2 นาที)
  5. ผู้ใช้กดปุ่ม "อัปเดตข้อมูลจาก Cloud" บนเว็บ Vercel เพื่อดึงข้อมูลใหม่มาแสดงผล


4. การดูแลรักษาระบบ (Maintenance & Environment Variables)
-----------------------------------
หากระบบมีปัญหา ให้ตรวจสอบความถูกต้องของรหัสลับ (Keys) เหล่านี้:
  - GIST_GITHUB_TOKEN : คือกุญแจสำคัญที่ใช้สั่งรันบอทและใช้อัปเดตไฟล์ (จำเป็นต้องตั้งค่าให้มีสิทธิ์ 'repo' และ 'gist') 
    ต้องใส่ไว้ 2 ที่คือ: Vercel (Environment Variables) และ GitHub (Repository Secrets)

  - NEXT_PUBLIC_GIST_ID : คือที่อยู่ของ Database 
    ต้องใส่ไว้ 2 ที่คือ: Vercel (Environment Variables) และ GitHub (Repository Secrets)
